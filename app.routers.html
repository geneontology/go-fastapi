async app.routers.bioentity.get_annotations_by_gene_id(id: str = Path(Ellipsis), slim: List[str] = Query(None), start: int = Query(0), rows: int = Query(None))

Returns GO terms associated with a gene.

Retrieves Gene Ontology (GO) terms associated with a gene identified by its CURIE identifier. The gene identifier should be represented in CURIE format (e.g., ZFIN:ZDB-GENE-050417-357).

Parameters:

  • id – The CURIE identifier of the gene for which GO term associations are retrieved. (required)

  • slim – Map objects up slim to a higher-level category. Value can be an ontology class ID or subset ID. Default is None. (optional)

  • start – The starting index for pagination. Default is 0. (optional)

  • rows – The number of results per page. Default is 100. (optional)

Returns:

A dictionary containing the GO term associations for the provided gene. The dictionary will contain fields such as ‘numFound’ and ‘associations’ associated with the gene.

IMPLEMENTATION DETAILS

Note: This method is implemented as a query to the GO/AmiGO Solr instance. The supported gene IDs include:

  • ZFIN (e.g., ZFIN:ZDB-GENE-050417-357)

Note that the AmiGO GOlr natively stores MGI annotations to MGI:MGI:nn. However, the standard for biolink is MGI:nnnn, so you should use this (will be transparently mapped to legacy ID).

Additionally, for some species such as Human, GO has the annotation attached to the UniProt ID. Again, this should be transparently handled; e.g., you can use NCBIGene:6469, and this will be mapped behind the scenes for querying.

async app.routers.bioentity.get_annotations_by_goterm_id(id: str = Path(Ellipsis), evidence: List[str] = Query(None), start: int = Query(0), rows: int = Query(None))

Returns annotations using the provided GO term.

Retrieves annotations based on the provided Gene Ontology (GO) term identifier. The GO term identifier should be represented in CURIE format (e.g., GO:0044598).

Parameters:

  • id – The CURIE identifier of the GO term to be used for annotation retrieval. (required)

  • evidence – List of evidence codes to filter the results. Default is None. (optional)

  • start – The starting index for pagination. Default is 0. (optional)

  • rows – The number of results per page. Default is 100. (optional)

Returns:

A dictionary containing the annotation information retrieved from the database. The dictionary will contain fields such as ‘date’, ‘assigned_by’, ‘bioentity_label’, ‘bioentity_name’, ‘synonym’, ‘taxon’, ‘taxon_label’, ‘panther_family’, ‘panther_family_label’, ‘evidence’, ‘evidence_type’, ‘reference’, ‘annotation_extension_class’, and ‘annotation_extension_class_label’ associated with the provided GO term.

Return type:

dict

Note:

  • For example, to get annotations for the GO term ‘GO:0044598’, the URL should be: ‘/api/bioentity/function/GO:0044598’.

  • The ‘evidence’ parameter can be used to filter annotations by specific evidence codes.

  • The ‘start’ and ‘rows’ parameters can be used for pagination of results. ‘start’ determines the starting index for fetching results, and ‘rows’ specifies the number of results to be retrieved per page.

async app.routers.bioentity.get_bioentity_by_id(id: str = Path(Ellipsis), start: int = Query(0), rows: int = Query(None))

Get bio-entities (genes) by their identifiers.

Retrieves bio-entities (genes) based on their identifiers in CURIE format.

Parameters:

  • id – The CURIE identifier of the bioentity to be retrieved. (required)

  • start – The starting index for pagination. Default is 0. (optional)

  • rows – The number of results per page. Default is 100. (optional)

Returns:

A dictionary containing the bioentity information retrieved from the database. The dictionary will contain fields such as ‘id’, ‘bioentity_name’, ‘synonym’, ‘taxon’, and ‘taxon_label’ associated with the specified bioentity.

Raises:

HTTPException – If the bioentity with the provided identifier is not found in the database.

Note:

  • For example, to get a gene with the identifier ‘ZFIN:ZDB-GENE-990415-44’, the URL should be: ‘/api/bioentity/ZFIN:ZDB-GENE-990415-44’.

  • The ‘start’ and ‘rows’ parameters can be used for pagination of results. ‘start’ determines the starting index for fetching results, and ‘rows’ specifies the number of results to be retrieved per page.

  • slim – Map objects up slim to a higher-level category. Value can be an ontology class ID or subset ID. Default is None. (optional)

  • start – The starting index for pagination. Default is 0. (optional)

  • rows – The number of results per page. Default is 100. (optional)

Returns:

A dictionary containing the gene annotation information retrieved from the database. The dictionary will contain fields such as ‘date’, ‘assigned_by’, ‘bioentity_label’, ‘bioentity_name’, ‘synonym’, ‘taxon’, ‘taxon_label’, ‘panther_family’, ‘panther_family_label’, ‘evidence’, ‘evidence_type’, ‘reference’, ‘annotation_extension_class’, and ‘annotation_extension_class_label’ associated with the provided GO term.
